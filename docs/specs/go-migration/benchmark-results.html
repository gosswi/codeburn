Key findings

Go vs installed

Go beats the uncached TypeScript v0.5.0 by 2.4-3.9x without requiring any warmup. The speedup grows with data volume because Go's parallel goroutine pool scales where Node.js's single-threaded parser does not.

Why TS Phase 2 still leads Go

modernc.org/sqlite (pure Go) has ~5-10x higher per-query overhead than better-sqlite3 (native C). The cache lookup cost neutralizes Go's parse speedup, pinning Go at a stable ~660ms even after warm runs. Replacing with CGO sqlite or mattn/go-sqlite3 would close this gap.

Memory: both optimized variants win

Both Go and TS Phase 2 use ~100MB RSS, compared to 187-336MB for the installed version. The V8 heap is not the culprit - it's the parsed JSON objects held in memory during aggregation. Go's struct allocations and TS Phase 2's smaller cache result set both reduce peak footprint by 44-70%.
